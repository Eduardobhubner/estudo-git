Informações iniciais sobre o git

Para se iniciar um reposotório no git, basta entrar na pasta desejada e digitar "git init". Um repositório é muito importante quando você quer determinar quais mudanças ocorreram no meio do caminho, quem alterou, quais foram todas as alterações, etc

Para saber o status do seu repositório git, digite a função "git status"

Sempre que baixar o git em seu device, temq ue configurar de quem é o repositório, para configurar basta " (git config --local user.name "...") e (git config --local user.email "...@gmail.com"

Sempre que quiser criar um commit, para poder gerar um "ponto" na linha de desenvolvimento de seu projéto, deve antes adicionar o arquivo que deseja que seja monitorado pelo repositório com o comando "git add 'name-file' e depois para criar o commit digitar "git commit -m'paragrafo para descrever as alterações' "

Para poder começar a trabalhar em equipe ou sozinho em máquinas diferentes, podemos tanto criar um servidor local como servidores remotos, para criar um servidor local, deve-se criar uma pasta, por exemplo: "servidor", e nele inicializar um repositório já mencionado acima, mas quando se trata de um servidor onde apenas vai ter a função de armazenar os arquivos, o git já fornece uma função especifica para isso( git init --bare )

Para criar servidores remotos, precisamos criar um repositório em versionadores de código git, como por exemplo o github, um dos mais famosos atualmente 24/05/2021, após a conta criada, devemos criar um repositório e puxar seu https para direcionar ao git em seu device aonde você deseja criar seu "servidor", para isso devemos usar a função (git remote add origin "url")

Mas vale lembrar que direcionar o acesso remoto não é necessariamente já colocar ou puxar arquivos de seu servidor, para enviar os commits deve usar a função ( git push master origin ) e para puxar a ultima versão do código ( git pull origin master

Branches ("ramos") são utilizados para desenvolver funcionalidades isoladas umas das outras. A branch master é a branch "padrão" quando você cria um repositório. É interessante separar o desenvolvimento de funcionalidades em branches diferentes, para que as mudanças no código para uma não influencie no funcionamento de outra.

Alternativa correta! Com o git checkout nós desfazemos uma alteração que ainda não foi adicionada ao index ou stage, ou seja, antes do git add. Depois de adicionar com git add, para desfazer uma alteração, precisamos tirá-la deste estado, com git reset. Agora, se já realizamos o commit, o comando git revert pode nos salvar.
